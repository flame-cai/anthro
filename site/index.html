Child Growth Calculator

Enter a child's measurements to get WHO 2006 nutritional status for 6 indicators. Only sex and date of birth are required — any missing measurement shows a clear explanation rather than an empty result.

Browser / CDN

Load two scripts. No build step, no npm. Tables are fetched once and cached by the browser.

<!-- library (~11 KB minified) -->
<script src="https://flame-cai.github.io/anthro/anthro.bundle.js"></script>
<!-- API helper: loads tables, exposes AnthroAPI -->
<script src="https://flame-cai.github.io/anthro/api.js"></script>
<script>
  await AnthroAPI.ready()

  const r = AnthroAPI.compute({
    mode:      'day',         // 'day' (default) | 'month'
    sex:       'female',
    dob:       '2024-01-15', // date of birth — most precise
    weight_kg: 7.0,
    height_cm: 64.0,
    muac_mm:   136,
  })

  r.lhfa           // 'Moderately stunted'
  r.z_lhfa         // -2.7901
  r.wfa            // 'Normal'
  r.muac_threshold // 'Normal'
  r.bmi_val        // 17.0898  (kg/m²)
</script>

The tables (~480 KB total) are served from flame-cai.github.io/anthro/data/ with Access-Control-Allow-Origin: *. You can fetch() them from any domain.

npm

npm install @flame-cai/anthro
const { createAnthro } = require('@flame-cai/anthro')
const fs = require('fs')

const names  = ['wfa','lhfa','bmi','acfa','wfl','wfh']
const load   = (p, n) => JSON.parse(fs.readFileSync(
  require.resolve(`@flame-cai/anthro/data/${p}_${n}.json`)))
const anthro = createAnthro(
  Object.fromEntries(names.map(n => [n, load('day',   n)])),
  Object.fromEntries(names.map(n => [n, load('month', n)]))
)

const result = anthro.compute({
  mode: 'day', sex: 2, dob: '2024-01-15',
  weight_kg: 7, height_cm: 64, muac_mm: 136,
})

Clone directly

Clone the repo and point to src/anthro.js and data/ directly. No npm, no build step. Works in any Node.js project, Electron app, or bundler.

git clone https://github.com/flame-cai/anthro.git
const { createAnthro } = require('./anthro/src/anthro.js')
const names  = ['wfa','lhfa','bmi','acfa','wfl','wfh']
const anthro = createAnthro(
  Object.fromEntries(names.map(n => [n, require(`./anthro/data/day_${n}.json`)])),
  Object.fromEntries(names.map(n => [n, require(`./anthro/data/month_${n}.json`)]))
)

Option B — copy files

Copy src/anthro.js and the entire data/ folder into your project.

your-rn-app/
  src/lib/
    anthro.js         ← src/anthro.js
    data/
      day_wfa.json    ← all 12 JSON files from data/
      ...

Important: call createAnthro() once at module level, not inside a component. The library pre-builds lookup maps at instantiation (~50 ms on a mid-range device).

// module-level singleton — do this
const anthro = createAnthro(dayTables, monthTables)
export default anthro

// not inside a component — avoid this
function MyComponent() {
  const anthro = createAnthro(...)  // rebuilds maps on every render
}

Missing inputs

Any missing measurement returns a descriptive string instead of null. Other indicators are still computed normally.

anthro.compute({ sex: 'F', dob: '2024-01-15', height_cm: 64 })

r.lhfa           // 'Moderately stunted'        ← computed
r.wfa            // 'Missing weight to compute'
r.wflh           // 'Missing weight to compute'
r.muac_threshold // 'Missing muac to compute'
r.acfa           // 'Missing muac to compute'

Table sources

Day-indexed tables are extracted from the R package anthro v1.1.0 (WorldHealthOrganization/anthro): weianthro, lenanthro, bmianthro, acanthro, wflanthro, wfhanthro. Same datasets as WHO igrowup SAS/SPSS/Stata software. Month-indexed tables are the WHO supplementary monthly tables from the same study.

Formula

Source: WHO Technical Report 2006, §5.2, pp. 300–304.

z = [(X/M)^L − 1] / (L × S)   when L ≠ 0
z = ln(X/M) / S                 when L = 0

SD23 adjustment when |z| > 3:
  z = +3 + (X − SD3) / (SD3 − SD2)    when z > +3
  z = −3 + (X − SD3') / (SD2' − SD3') when z < −3
  SD3 = M(1 + L·S·3)^(1/L),  SD2 = M(1 + L·S·2)^(1/L)

Measure correction

Source: WHO 2006, Chapter 7. Average recumbent–standing difference = 0.73 cm, applied as 0.7 cm.

Table expectsChild measured asCorrection
L (recumbent)H (standing)height + 0.7 cm
H (standing)L (recumbent)length − 0.7 cm
SameNone

In day mode, the loh field from lenanthro per day determines the expected type (L for days 0–730, H for 731–1826). In month mode: L for months 0–23, H for months 24–60.

Plausibility flags

Source: WHO igrowup documentation (igrowup_standard.sas). Flag = 1 means the z-score is outside the biologically plausible range and should be reviewed.

Precision note

This library uses the exact floating-point z-score for classification. R anthro rounds z to 2 decimal places before classifying — z = −2.003 becomes −2.00 and is classified as Normal instead of MAM. These borderline cases are clinically important, so we do not round.

Changelog

1.0.0

  • Day-indexed (WHO igrowup) and month-indexed (WHO supplementary) table modes
  • Exact floating-point z-scores — no pre-rounding before classification
  • Automatic ±0.7 cm measure correction from loh field
  • 6 indicators: muac_threshold, acfa, bmi, lhfa, wfa, wflh
  • Missing input → descriptive string, not null
  • Day 1826 (exactly 60 months) inclusive in all tables
  • TypeScript declarations
  • 33 tests verified against R anthro v1.1.0
  • Works in Node.js, browser, React Native

Contributing

npm run deploy   # builds then pushes to gh-pages branch
