Usage

  
anthro v1 works in every JavaScript environment. Choose your integration method below.


  

    
    
  


  

    
Browser CDN

    
Load two scripts and all tables are fetched automatically. No build step required.

    

      

<!-- 1. Load the library bundle (~11 KB minified) -->
<script src="https://flame-cai.github.io/anthro/anthro.bundle.js"></script>

<!-- 2. Load the API helper (handles table fetching + caching) -->
<script src="https://flame-cai.github.io/anthro/api.js"></script>

<script>
  // 3. Wait for tables to load (cached after first visit)
  await AnthroAPI.ready();

  // 4. Compute all 6 indicators
  const result = AnthroAPI.compute({
    mode:      'day',        // 'day' (default) | 'month'
    sex:       'female',    // 'male'|'female'|'m'|'f'|'M'|'F'|1|2
    dob:       '2024-01-15',// date of birth — most precise
    weight_kg: 7.0,          // or weight_g: 7000
    height_cm: 64.0,         // 0.5 cm steps
    muac_mm:   136,          // or muac_cm: 13.6
  });

  console.log(result.lhfa);            // 'Moderately stunted'
  console.log(result.z_lhfa);          // -2.7901
  console.log(result.wfa);             // 'Normal'
  console.log(result.muac_threshold);  // 'Normal'
</script>

    


    
Missing inputs

    
If any measurement is absent, the relevant indicator returns a descriptive string — never a silent null.

    

      

AnthroAPI.compute({ sex: 'F', dob: '2024-01-15', height_cm: 64 })
// result.wfa  → "Missing weight to compute"
// result.wflh → "Missing weight to compute"
// result.lhfa → 'Moderately stunted'   ← computed normally
// result.muac_threshold → "Missing muac to compute"

    

  


  

    
npm / Node.js

    

      

npm install @flame-cai/anthro

    

    

      

const { createAnthro } = require('@flame-cai/anthro')
const fs = require('fs')

// Load day-indexed and month-indexed tables
const names = ['wfa', 'lhfa', 'bmi', 'acfa', 'wfl', 'wfh']
const load  = (prefix, name) =>
  JSON.parse(fs.readFileSync(`data/${prefix}_${name}.json`))

const day   = Object.fromEntries(names.map(n => [n, load('day',   n)]))
const month = Object.fromEntries(names.map(n => [n, load('month', n)]))
const anthro = createAnthro(day, month)

// Compute — day mode (DOB gives exact age in days)
const result = anthro.compute({
  mode:      'day',
  sex:       2,            // 1=male, 2=female
  dob:       '2024-01-15',
  weight_kg: 7.0,
  height_cm: 64.0,
  muac_mm:   136,
})

// Batch — mixed modes per row
const results = anthro.batch([
  { mode: 'day',   sex: 'M', dob: '2023-06-01', weight_kg: 10.5, height_cm: 77 },
  { mode: 'month', sex: 'F', age_months: 9,  weight_kg: 7.0,  height_cm: 64 },
])

Documentation


  
Table sources


  

    
Day-indexed tables (mode: 'day') — default

    
Extracted from the WHO-maintained R package anthro v1.1.0 (WorldHealthOrganization/anthro), which redistributes the official WHO igrowup day-indexed datasets (weianthro, lenanthro, bmianthro, acanthro, wflanthro, wfhanthro).

    
One LMS row per day (age 0–1826 days, birth to 60 months inclusive); height tables in 0.1 cm steps. Same tables as WHO igrowup SAS/SPSS/Stata/S-Plus software packages.

    
Use when: exact date of birth is available — eliminates rounding error from month conversion.

  


  

    
Month-indexed tables (mode: 'month')

    
WHO-published supplementary monthly LMS tables, same 2006 Multicentre Growth Reference Study. One LMS row per whole month (0–60 months); height in 0.5 cm steps.

    
Use when: only completed months are available, or when results must match Excel/spreadsheet-based workflows.

  


  
Formula

  
Restricted LMS (Box-Cox power transformation) — WHO Technical Report 2006, §5.2, pp. 300–304:


  


z = [(X/M)^L − 1] / (L × S)   when L ≠ 0
z = ln(X/M) / S                 when L = 0

// SD23 adjustment beyond ±3 SD (restricted LMS):
z_adj = +3 + (X − SD3) / (SD3 − SD2)    when z > +3
z_adj = −3 + (X − SD3') / (SD2' − SD3') when z < −3

// Where SD3 = M(1 + L·S·3)^(1/L),  SD2 = M(1 + L·S·2)^(1/L)

  

  
This library uses the exact floating-point z-score for classification. The R anthro package rounds z to 2 decimal places before classifying; this loses precision for children exactly at a cut-point boundary (e.g. z = −2.003 rounds to −2.00 and would be misclassified as Normal).


  
Measurement correction (±0.7 cm)

  
Source: WHO MGRS (2006), Chapter 7. Average recumbent–standing difference = 0.73 cm, applied as 0.7 cm.

L (recumbent)H (standing)height + 0.7 cm before LHFA/BMI lookup

        
H (standing)L (recumbent)length − 0.7 cm before LHFA/BMI lookup

        
Same as measured—No correction

      
    

  

  
In day mode, the loh field from the lenanthro table per day determines the expected measure type. In month mode: L expected for months 0–23, H for months 24–60.
