<script src="https://flame-cai.github.io/anthro/api.js"></script>
<script>
  await AnthroAPI.ready()   // loads tables once, cached

  const r = AnthroAPI.compute({
    mode: 'day',           // 'day' | 'month'
    sex:  'female',        // or 'F' | 2
    dob:  '2024-01-15',    // most accurate
    weight_kg: 7.0,
    height_cm: 64.0,
    muac_mm:   136,
  })

  r.lhfa            // 'Moderately stunted'
  r.z_lhfa          // -2.7901
  r.wfa             // 'Normal'
  r.muac_threshold  // 'Normal'
  r.wflh            // 'Normal'
  r.acfa            // 'Normal'
  r.bmi             // 'Normal'
</script>

Method 2 — npm (Node / bundlers)

npm install @flame-cai/anthro

const {createAnthro} = require('@flame-cai/anthro')
const fs = require('fs')
const load = (p,n) =>
  JSON.parse(fs.readFileSync(`data/${p}_${n}.json`))
const names = ['wfa','lhfa','bmi','acfa','wfl','wfh']
const day   = Object.fromEntries(names.map(n=>[n,load('day',n)]))
const month = Object.fromEntries(names.map(n=>[n,load('month',n)]))
const anthro = createAnthro(day, month)

const result = anthro.compute({
  mode: 'day', sex: 2, age_days: 289,
  weight_kg: 7, height_cm: 64, muac_mm: 136
})

Method Comparison

n = 133 children (AIMM pilot data). All 10 pairwise combinations of 5 methods across 5 z-score indicators.

Why are 5 pairs not exactly 100%?

Every non-100% case has been forensically investigated. There are no errors in this library. All mismatches are caused by behaviour in R anthro, not in our implementation:

Cases 1–2 (Our Month vs R Month, LHFA, age=24m)
R anthro converts 24 months → day 730 (= round(24 × 30.4375)). Day 730 is the exact loh transition boundary (L→H). At that day, R sees loh='L' and measure='H', applies +0.7 cm recumbent correction (h=78.7 cm), giving z=−2.99 (Moderately stunted). Our month-24 table sets expected measure = 'H' (month ≥ 24), so no correction, giving z=−3.21 (Severely stunted). The children's actual age in days was 746 and 751 (both loh='H', no correction needed). Our day-mode answer is correct for these children's actual ages.

Cases 3–5 (R anthro output rounding)
R anthro rounds z-scores to 2 decimal places before classifying. When the exact z is −2.0025, −2.0035, or −3.0039, R rounds to −2.00 or −3.00 and classifies as Normal or MAM respectively (since −2.0 is not < −2). Our library uses the full floating-point z-score for classification — this is mathematically correct per WHO (2006) §5.2 which states z < −2 is the cut-point. z = −2.0025 IS < −2; R anthro's rounding loses this precision.

Data Sources & Citations

Table source (day-indexed — default mode)

This library uses the official WHO igrowup day-indexed LMS tables: weianthro, lenanthro, bmianthro, acanthro, wflanthro, wfhanthro. One LMS row per day (0–1826 d) for age-based indicators; one row per 0.1 cm for height-based. These are the same tables used by WHO igrowup SAS, SPSS, Stata, and S-Plus software packages, and embedded in the WHO-maintained R package anthro v1.1.0.

Month-indexed tables (mode:'month') are the WHO supplementary monthly tables, same 2006 study, one row per whole month (0–60 m), heights in 0.5 cm steps.
