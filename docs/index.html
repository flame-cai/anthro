Browser / CDN

        
Add two script tags. No build step, no dependencies. Tables are fetched once and cached by the browser.


<!-- Library bundle (~11 KB) -->
<script src="https://flame-cai.github.io/anthro/anthro.bundle.js"></script>
<!-- API helper: loads tables, exposes AnthroAPI -->
<script src="https://flame-cai.github.io/anthro/api.js"></script>
<script>
  await AnthroAPI.ready()   // fetches all 12 JSON tables, then caches them

  const r = AnthroAPI.compute({
    mode:      'day',        // 'day' (default) | 'month'
    sex:       'female',    // 'male'|'female'|'m'|'f'|'M'|'F'|1|2
    dob:       '2024-01-15',// date of birth — exact age in days
    weight_kg: 7.0,
    height_cm: 64.0,
    muac_mm:   136,
  })

  r.lhfa            // 'Moderately stunted'
  r.z_lhfa          // -2.7901
  r.wfa             // 'Normal'
  r.muac_threshold  // 'Normal'
  r.bmi_val         // 17.0898 (kg/m²)
</script>

Clone directly

        
Clone the repository and use src/anthro.js and data/ directly in any project, without npm. This works in Node.js, React Native, Electron, and any bundler (webpack, Vite, Metro).


git clone https://github.com/flame-cai/anthro.git


// From your project root, assuming anthro/ is cloned alongside it:
const { createAnthro } = require('./anthro/src/anthro.js')
const names = ['wfa','lhfa','bmi','acfa','wfl','wfh']
const anthro = createAnthro(
  Object.fromEntries(names.map(n => [n, require(`./anthro/data/day_${n}.json`)])),
  Object.fromEntries(names.map(n => [n, require(`./anthro/data/month_${n}.json`)]))
)

        
No build step required. src/anthro.js has no external dependencies and uses only standard JavaScript.

      


      
      

        
React Native (Android & iOS)

        
The library works in React Native without modification. The Metro bundler handles require() for JSON files natively.


        
Option A — npm install (recommended)


npm install @flame-cai/anthro


// In any component or service file:
import { createAnthro } from '@flame-cai/anthro'

// require() JSON assets — Metro bundles them at build time
const names = ['wfa','lhfa','bmi','acfa','wfl','wfh']
const day = Object.fromEntries(names.map(n =>
  [n, require(`@flame-cai/anthro/data/day_${n}.json`)]))
const month = Object.fromEntries(names.map(n =>
  [n, require(`@flame-cai/anthro/data/month_${n}.json`)]))

const anthro = createAnthro(day, month)  // create once, reuse everywhere

// Use in a component:
export function assessChild(childData) {
  return anthro.compute({
    mode:      'day',
    sex:       childData.sex,
    dob:       childData.dateOfBirth,   // ISO string e.g. '2023-06-01'
    weight_kg: childData.weightKg,
    height_cm: childData.heightCm,
    muac_mm:   childData.muacMm,
  })
}

Performance note

        
Create the anthro instance once (at module level or in a singleton), not inside a component render. The library pre-builds O(1) lookup maps at instantiation — this takes ~50 ms on a mid-range device and should not happen on every render.


// ✓ Good — module-level singleton
const anthroInstance = createAnthro(dayTables, monthTables)
export default anthroInstance

// ✗ Avoid — creates and discards instance on every render
function MyComponent() {
  const anthro = createAnthro(...) // slow!
}

      


      
      

        
Missing inputs

        
If any measurement is missing, the affected indicator returns a descriptive string — not null, not an error. Other indicators are still computed.


anthro.compute({ sex: 'F', dob: '2024-01-15', height_cm: 64 })

// result.lhfa  → 'Moderately stunted'        ✓ computed normally
// result.wfa   → 'Missing weight to compute'  ← descriptive
// result.wflh  → 'Missing weight to compute'
// result.muac_threshold → 'Missing muac to compute'
// result.acfa  → 'Missing muac to compute'

Day-indexed tables are extracted from the R package anthro v1.1.0 (WorldHealthOrganization/anthro), which redistributes the WHO igrowup datasets: weianthro, lenanthro, bmianthro, acanthro, wflanthro, wfhanthro. These are the same datasets used in the WHO igrowup SAS/SPSS/Stata packages.

        
Month-indexed tables are the WHO-published supplementary monthly LMS tables from the same 2006 study.

      


      
      

        
Formula

        
Source: WHO MGRS (2006), Technical Report §5.2, pp. 300–304.


z = [(X/M)^L − 1] / (L × S)   when L ≠ 0
z = ln(X/M) / S                 when L = 0

SD23 adjustment (restricted LMS — for |z| > 3):
  z = +3 + (X − SD3) / (SD3 − SD2)   when z > +3
  z = −3 + (X − SD3') / (SD2' − SD3') when z < −3

  where  SD3 = M(1 + L·S·3)^(1/L)
         SD2 = M(1 + L·S·2)^(1/L)

      


      
      

        
Measure correction

        
Source: WHO MGRS (2006), Chapter 7. Average recumbent–standing difference = 0.73 cm, applied as 0.7 cm.

        

          
Table expectsMeasured asCorrection

          
            
L (recumbent)H (standing)height + 0.7 cm

            
H (standing)L (recumbent)length − 0.7 cm

            
Same as measuredNone

          
        

        
In day mode, the loh field from lenanthro per day determines the expected measure type (L for days 0–730, H for 731–1826). In month mode: L for months 0–23, H for months 24–60.

Precision note

        
This library uses the exact floating-point z-score for classification, without rounding before the cut-point comparison. The R anthro package rounds z to 2 decimal places before classifying, which causes misclassification for children with z exactly at a boundary (e.g. z = −2.003 rounds to −2.00 and is classified Normal instead of MAM).

        
In practice this affects only children very close to a cut-point boundary — but these are the clinically important borderline cases where precision matters most.

Contributing

        
All contributions are welcome.

        
Tests

        
node test/anthro.test.js — no test framework, no dependencies.

        
Pre-commit hook (Husky)

        
After cloning, run npm install. This activates a Husky pre-commit hook that runs the test suite before every git commit. Commits are blocked if any test fails.

        
Updating WHO tables

        
Day-indexed tables are extracted from the R anthro package. To regenerate after a WHO update:

        
Rscript scripts/export-tables.R
node test/anthro.test.js

        
Month-indexed tables (data/month_*.json) are maintained manually from the WHO website.
